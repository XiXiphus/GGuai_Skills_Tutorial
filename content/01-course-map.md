# Course Map

## Course Thesis

这不是一套“记住所有命令”的课，而是一套“学会把 skill 当成稳定工法”的课。

课程的主线是：

1. 先知道 `skill` 是什么。
2. 再知道怎么开口，怎么把模糊需求说清。
3. 再掌握三把最有杠杆的基础工具。
4. 然后学会把材料变成理解，把理解变成表达，把表达变成成品。
5. 最后把多个 skill 串成自己的 workflow。

## Why This Order

- 如果一开始就讲具体 skill，她会把它们当“神奇命令”。
- 如果不先讲怎么开口，她后面调 skill 也容易越调越歪。
- 如果不会先降秩、先说人话，后面的写作和卡片都只是包装。
- 如果没有 workflow 视角，她会觉得每个 skill 是孤岛。

## Modules

### 1. What Is A Skill

目标：建立正确心智模型。

她需要明白：skill 不是咒语，不是“更高级的 prompt”，而是把 Claude 拉进某种稳定工作方式的工法。

### 2. Ask Well

目标：学会开一个靠谱的头。

她需要明白：材料、目标、对象、限制、产物，这五件事说清，Claude 才好发力。

### 3. First Three Skills

目标：先掌握最有杠杆的三把工具。

主角是：`ljg-rank`、`ljg-plain`、`ljg-learn`。

### 4. Material To Understanding

目标：学会处理不同输入。

文件、网址、粘贴文本、一个概念，这四种入口会让 skill 用法不同。

### 5. Writing For Someone

目标：把“我好像懂了”变成“我能讲给别人听”。

主角是：`ljg-writes`。

### 6. Card And Delivery

目标：把表达做成一张别人愿意看的卡。

主角是：`ljg-card`。

### 7. Workflows

目标：不再单点使用 skill，而是会串。

她需要拿到 2-3 条真正能反复使用的小工作流。

### 8. Capstone And Electives

目标：把这套东西变成她自己的。

这里会做一个毕业项目，并把高阶 skill 摆成选修地图，包括：

- `ljg-paper`
- `ljg-paper-river`
- `ljg-word`
- `ljg-travel`
- `ljg-relationship`
- `ljg-roundtable`

### 9. Final Reveal

目标：揭示课程的隐藏布线逻辑，完成从使用者到设计者的视角切换。

这不是一节"新技能课"，而是一次回望。让她意识到：那些她觉得"好舒服"的课程内容，本身就是用她刚学的 skills 生成的；她已经从"踩陷阱的人"变成了"会看见红线、甚至会自己布线的人"。课程在此真正结束。

## Graduation Standard

学完后，她不需要像“熟练用户”那样会一切。

她只需要做到：

- 遇到一个主题，知道先调哪把工具
- 能把一次理解过程变成一段能讲给人听的话
- 能把一段文字做成一张成品卡片
- 能挑出对自己最有用的下一组 skill
