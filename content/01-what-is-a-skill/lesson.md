# 第一课：Skill 是什么

## 开场：你可以先把它想成拍照

你手机里一定有很多随手拍的照片。

下楼看到夕阳好看，掏出来按一张，这叫**随口问 Claude**——轻快、即兴，拍完就完。

但如果今天要拍一组生日照，你可能会打开相机的"人像模式"，因为它会帮你自动虚化背景、调肤色、补光。你不用自己一条条参数去抠，模式已经帮你把一套成熟的做法装进去了。这就像是**调一个 skill**。

而如果你要策划一次三天旅行，想拍 vlog、拍美食、拍风景、最后拼成一支完整的片子，你就会按顺序调用好几个模式，再剪辑在一起。这就叫**把 skill 串成 workflow**。

Skill 本身没什么神秘的。它更像是一种"工作方式"的 shortcut：当你发现某件事总得按固定套路做，就把它交给 Claude 里对应的 skill，减少你从零解释的力气。

---

## 三个层次，到底是什么感觉

### 第一层：随口问 Claude

你想到什么就问什么。

> "这段英文帮我翻译一下。"
> "这个词什么意思？"
> "帮我写个请假理由。"

好处是快，坏处是结果忽高忽低。今天翻译得顺，明天就可能太正式、太生硬。因为它完全取决于你当时提问的方式，Claude 也只能凭经验猜你想要的风格。

### 第二层：调一个 skill

当你想要的是**某种特定处理结果**，而不是一个"差不多就行"的答案，就该想到 skill。

看三个轻量的例子：

**例子 1：你看到一篇行业报告，术语满天飞，想发给完全外行的朋友看。**

随口问可能得到一份"精简版"，但还是硬邦邦的。如果你调 `ljg-plain`，它的工作方式本来就是"把复杂内容讲成普通人能懂的话"——你会得到一份更像"朋友口述"的版本。

**例子 2：你想研究"为什么现在的年轻人不爱换手机了"，刷到一堆零散分析，脑子很乱。**

随口问 Claude 可能给你罗列十条原因，但你还是抓不到重点。如果你调 `ljg-rank`，它的工法是"把现象砍到最不可再少的几根支柱"——它会帮你找出真正撑着这个问题的 3-4 个核心力量，而不是把所有现象堆在一起。

**例子 3：你读完一本书，想把一个打动你的观点做成可以发朋友圈的图。**

随口问 Claude"帮我写段文案"，它可能写得很长、格式不对、也不像卡片。如果你调 `ljg-card`，它会直接按做卡片的逻辑来组织文字：句子短、有重点、适合视觉呈现。

### 第三层：把多个 skill 串成 workflow

有时候一件事不是一步能搞定的。

比如你读到一篇很难的论文，想最后做成一张科普卡片发朋友圈。那你的 workflow 可能是：

1. 先用 `ljg-rank` 把论文的核心论点拆成骨架；
2. 再用 `ljg-plain` 把骨架翻译成大白话；
3. 最后用 `ljg-card` 把这段大白话变成一张图。

这就不是"调一个命令"了，而是你心里有了一条**从小路到大道的高速公路**。

## Skill 的真正价值：减少来回试错的次数

很多人觉得 skill"很酷"或者"很高级"。其实不是。

它最大的价值是**省口水、稳预期**。你不用再从零跟 Claude 解释"我要你简短一点、口语一点、分点来说"——skill 已经把这些偏好内化了。你就像是请了一位已经熟悉你风格的助手，而不是每次都要重新培训。

当你越常处理某类任务，就越会知道：这件事，下次直接用 skill 就好了。

---

## 什么时候该想到"要不要调个 skill"

这里给你一个轻量判断清单。如果下面任何一条中了，就值得想一想：

- [ ] 这个问题我上周好像也以类似方式问过 Claude。
- [ ] 我想要的不只是"答案"，而是一种**处理后的形态**（简化版、骨架版、卡片版）。
- [ ] 我预感要跟 Claude 来回三四轮才能说清楚要求。
- [ ] 我想把这件事变成一个以后能重复用的做法。

不用全中，中一条就够了。

---

## 本课小结

Skill 不是咒语，不是更厉害的 prompt 模板。

它是：**把 Claude 拉进一种你已经验证过的、稳定工作方式的工法。**

随口问是散步，调 skill 是开车走高速，串 workflow 是一次完整的旅行规划。后面我们要学的，就是怎么看清路况、选对高速、偶尔自己拼一条路线。
