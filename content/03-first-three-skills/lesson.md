# 第三课：三把第一梯队工具

这节课我们要认识三把最趁手的基础工具。它们不是功能类似的替代品，而是三种完全不同的"看问题方式"。

我用一个你绝对熟悉的主题来串讲：**拖延**。

---

## 第一把刀：ljg-rank —— 先找骨架

### 它在看什么

`ljg-rank` 不是帮你总结"拖延的五个原因"，而是在问：如果要把"拖延"这个现象砍到不能再砍，背后到底是哪几股独立的力在撑着她？

它要的是**生成器**，不是**标签**。

比如 `ljg-rank` 可能会告诉你，拖延不是"懒+完美主义+时间管理差"这三个并列的毛病，而是两股更底层的力在打架：

- 一股力是"当下自我的保护机制"（此刻舒服很重要）
- 另一股力是"未来自我的想象投射"（明天的你会更有能量）

**拖，不是因为你差劲，是因为这两股力还没看清彼此。**

### 最适合什么时候用

- 面对一个复杂议题，感觉"因素太多理不清"
- 想找到那个"牵一发而动全身"的支点
- 不想被现成的分类框架绑架，想回到事情本身

### 最不适合什么时候用

- 只想快速了解一个陌生领域的基础知识
- 需要立即可执行的操作清单
- 情绪正在上头，需要先被理解而不是被分析

---

## 第二把刀：ljg-plain —— 讲人话

### 它在看什么

`ljg-plain` 不是在"简化"，而是在**去术语化**。它默认你其实懂了，只是被一套不属于自己的表达裹住了。

比如关于拖延，你可能读过很多文章都用"执行功能障碍""时间折扣偏好""自我损耗理论"这些词。`ljg-plain` 会帮你把这些翻译成：

> "大脑觉得此刻刷手机是确定的爽，而工作的回报又远又模糊，所以它选了确定的。"

这不是降智，这是**把伪装的复杂剥掉，露出真正的结构**。

### 最适合什么时候用

- 读了一篇学术味很重的文章，想确认自己是不是真的懂了
- 要把一个概念讲给完全没背景的人听
- 发现自己被术语绕晕，越读越没底气

### 最不适合什么时候用

- 需要保留精确术语的场景（比如写论文）
- 对方确实需要学这些术语的时候
- 问题本身还不清楚，需要先深挖而不是先翻译

---

## 第三把刀：ljg-learn —— 往下多挖一层

### 它在看什么

`ljg-learn` 不满足于"知道"，它要的是**理解之后的那个顿悟时刻**。

它用八把解剖刀（历史、辩证、现象学、语言学、形式化、存在主义、美学、元哲学）去撬开一个概念。对于拖延，它可能会带你发现：

- **历史上**：拖延曾是贵族的特权，"慢慢做"是身份象征，现代人却被迫"高效"
- **现象学**：拖延时你其实不是"没做"，而是在做"准备做"的动作，体验本身很饱满
- **语言学**：中文说"拖"是被动被拉，英文"procrastinate"来自"明天"，是主动推迟

这些不是为了堆知识，而是为了让你**对拖延有了一种新的感知方式**。

### 最适合什么时候用

- 对一个主题已经有基本了解，想让它"活"起来
- 准备做分享、写文章，需要一点别人没注意到的角度
- 感觉自己"知道但说不出来"，需要打通某个卡点

### 最不适合什么时候用

- 完全陌生的领域（先维基百科，再 `ljg-learn`）
- 时间紧迫，只需要够用就行的信息
- 情绪需要被安抚，而不是被刺激思考

---

## 三把刀的差别到底在哪

| 工具        | 核心动作 | 会让问题变得         |
| ----------- | -------- | -------------------- |
| `ljg-rank`  | 砍到骨架 | **更锋利**，抓住支点 |
| `ljg-plain` | 剥去术语 | **更透明**，没有障碍 |
| `ljg-learn` | 多挖一层 | **更立体**，有惊喜感 |

它们可以单独用，也可以串着用：

- 先 `rank` 找骨架 → 再 `plain` 讲清楚 → 最后 `learn` 挖惊喜
- 或者反过来：先 `learn` 打开视野 → 再 `rank` 理结构 → 最后 `plain` 做输出

没有固定顺序，但**不要同时调三个**，一次只用一把刀。

---

## "我现在该调哪把"判断表

| 你的状态                   | 先调哪把                  | 为什么                         |
| -------------------------- | ------------------------- | ------------------------------ |
| 信息太多，不知道从哪里下手 | `ljg-rank`                | 先砍到骨架，其他都是噪音       |
| 被术语绕晕，越读越没底气   | `ljg-plain`               | 先确认自己真的懂了             |
| 懂了但觉得无聊，想找到新意 | `ljg-learn`               | 往下多挖一层，找那个"啊哈"时刻 |
| 准备讲给别人听             | `ljg-plain` → `ljg-learn` | 先讲人话，再给惊喜             |
| 要写深度长文               | `ljg-rank` → `ljg-learn`  | 先立骨架，再填血肉             |
| 只是想快点搞定任务         | 先别调 skill              | 直接开干，工具不是借口         |

---

## 这节课的真正目的

不是让你记住三个命令。

是让你意识到：**以前你可能总是拿错刀**。

信息太多的时候，你不是去找骨架，而是去"再多读几篇"。术语太多的时候，你不是去翻译成人话，而是去"硬背下来假装懂"。懂了但无聊的时候，你不是去深挖，而是去"换个话题"。

三把工具，对应三种常见的卡壳。知道哪把刀切什么问题，比把每把刀都磨得锋利更重要。
