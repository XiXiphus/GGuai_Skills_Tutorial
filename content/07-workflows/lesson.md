# 第7课：把工作流变成你的

## 开篇：从点菜到套餐

想象一下去餐厅。你当然可以单点：要一个沙拉、一份汤、一份主食。但很多人其实更想点套餐——因为套餐是配好的，你不用每次都重新想"今天怎么搭"。

Skill 也一样。单独调一个 `ljg-learn` 很好，但你知道什么时候该学完立刻接 `writes`，什么时候该先做 `rank` 再决定要不要学——这就更好了。

这一课就是帮你配好三套"套餐"。以后遇到常见场景，你不用从零想，有路可走。

---

## 为什么 Workflow 比单点更稳

单点 skill 的问题是：你每次都要重新决定"接下来干嘛"。

比如学了一个概念，懂了，然后呢？是写成一段话，还是做成卡片，还是再往下挖一层？这个"然后"如果没想好，很容易停在"我好像懂了"但什么都没留下。

Workflow 解决的就是这个：它把"从 A 到 B 到 C"的路径定下来，你只需要判断"我现在在 A"，就知道该往哪走。

它不是束缚，是省脑子的默认值。你随时可以改，但先有一个默认值。

---

## Workflow 一：学一个概念

**触发场景**：你遇到一个词、一个理论、一个你感兴趣但不了解的东西。比如"stoicism"、"费曼技巧"、"内卷"。

**路径**：

```text
输入概念 → ljg-learn → 得到深层理解 → ljg-plain → 用自己的话讲一遍 → （可选）ljg-card
```

**为什么这样串**：

1. **先用 `ljg-learn`**：它帮你把概念拆成多维度，你会有"原来它和这几个东西有关"的惊喜。这是"挖下去"。
2. **再接 `ljg-plain`**：挖完容易掉进术语里，用它把自己拉回来，确保你能讲给没听过的人听。
3. **最后可选 `ljg-card`**：如果你觉得这个理解值得存下来或分享，就做成卡片。

**最终产物**：一段你能讲出来的、没有术语的理解，或一张卡片。

**最小例子**：

> 输入：我想懂"认知失调"。
>
> Step 1：`ljg-learn 认知失调` → 得到它的心理学背景、常见例子、和"自我合理化"的区别。
>
> Step 2：`ljg-plain 认知失调是指...` → 得到"就是当你做了和你信念不符的事，大脑为了给这件事找理由，可能会改变你的信念"。
>
> Step 3：如果觉得好，用 `ljg-card` 做成一张"认知失调：当你的行为改变了你的信念"的卡片。

## Workflow 二：吃掉一篇文章或材料

**触发场景**：你读了一篇文章、看了一个视频、听了一期播客，觉得有东西，但有点多，不知道怎么收。

**路径**：

```text
输入材料 → ljg-rank → 找出骨架/核心论点 → ljg-learn（对核心论点）→ 深入理解 → ljg-writes → 变成你能讲的话
```

**为什么这样串**：

1. **先用 `ljg-rank`**：材料太多的时候，先别急着全吃。用它找出"这到底在说什么"的那根主线。这是"先找骨架"。
2. **对骨架用 `ljg-learn`**：主线找到了，对它进行深层拆解，看看有没有什么反直觉的东西。
3. **用 `ljg-writes` 输出**：把理解变成一段有主语、有对象、有场景的话。不是摘要，是你能用的表达。

**最终产物**：一段你能直接用的转述，或一条朋友圈/聊天时能抛出来的观点。

**最小例子**：

> 输入：我读了一篇关于"远程工作为什么让人更累"的长文。
>
> Step 1：`ljg-rank` → 发现核心论点：不是工作量大，是"切换成本"被低估了——在家没有物理边界，大脑一直在"准备切换"。
>
> Step 2：`ljg-learn 切换成本` → 深入了解这个概念，发现它和意志力、决策疲劳都有关。
>
> Step 3：`ljg-writes` → 得到"远程工作累，不是因为事情多，是因为你的大脑从来没下班"这样的表达。

## Workflow 三：做一张可发出去的卡片

**触发场景**：你已经有点想法了，想把它做成一张好看、好懂、愿意分享的卡片。

**路径**：

```text
零散想法/已有理解 → ljg-writes → 整理成一段完整表达 → ljg-card → 成品
```

**为什么这样串**：

1. **先用 `ljg-writes`**：很多人直接做卡片会卡，因为想法是散的。用它先把"我到底想说什么"理清楚，得到一个完整的、有人味的段落。
2. **再用 `ljg-card`**：把这段文字变成视觉。它有几种模具可选，长卡、信息图、多卡等等。

**最终产物**：一张 PNG 卡片，你可以发朋友圈、存笔记、或发给朋友。

**最小例子**：

> 输入：我觉得"早起"这件事被过度神话了。
>
> Step 1：`ljg-writes 早起不是美德，是偏好。有人就是晚上效率高，强迫自己早起反而...` → 得到一段结构完整的话。
>
> Step 2：`ljg-card -l` → 得到一张长阅读卡片，标题、正文、小结都排好了。

## 三条路怎么选

| 如果你...                    | 走哪条                |
| ---------------------------- | --------------------- |
| 遇到一个想懂的概念           | Workflow 一（学概念） |
| 读完/听完一份材料，想收下来  | Workflow 二（吃材料） |
| 已经有想法，想做成能发的卡片 | Workflow 三（做卡片） |

其实还有第四种情况：你也不知道自己在哪。那就先用 `ljg-rank` 看看"我现在手里到底是什么"。

---

## 确认理解

问自己这两个问题：

1. 下次你朋友问你"最近在学什么"，你能用哪条 workflow 把你的学习过程讲清楚？
2. 如果你读完一篇文章觉得"说得对但太长"，第一步该调哪个 skill，为什么？

---

## 这一课的真正收获

你不是学会了三条"必须遵守的流程"。

你是学会了：原来这些 skill 可以这样搭。以后你也可以自己配新的 workflow。

比如你可能发现："我看完电影想写短评， Workflow 二好像能用，但最后一步我想换成别的东西。" 那就改。Workflow 是你的，不是 Claude 的。

重点是，你现在有默认值了。遇事不用从零想。
