Lesson 01

Skill 初识 · 何为技能

不是咒语，也不是「更高级的 prompt」。
是让 Agent 跟着一套稳当的 habit 来干活。

开场：你可以先把它想成拍照

你手机里一定有很多随手拍的照片。

下楼看到夕阳好看，掏出来按一张，这叫随口丢一句——轻快、即兴，拍完就完。

但如果今天要拍一组生日照，你可能会打开相机的「人像模式」，因为它会帮你自动虚化背景、调肤色、补光。你不用自己一条条参数去抠，mode 已经帮你把一套成熟的做法装进去了。这就像是调一个 Skill。

而如果你要策划一次三天旅行，想拍 vlog、拍美食、拍风景、最后拼成一支完整的片子，你就会按顺序调用好几个 mode，再剪辑在一起。这就叫把好几个 Skills 串成一条 Workflow。

Skill 本身没什么神秘的。它更像是一种「省事捷径」：当你发现某件事总得按固定套路做，就把它交给 Agent，让它用对应的 Skill 来处理，少费口舌、少从头解释。

（不少产品里写作 Skill，说的是一回事。）

三个层次，到底是什么感觉

第一层：随口丢一句（Casual Ask）

你想到什么就问什么。

  • 「这段英文帮我翻译一下。」
  • 「这个词什么意思？」
  • 「帮我写个请假理由。」

好处是快，坏处是结果忽高忽低。今天翻译得顺，明天就可能太 formal、太生硬。因为它完全取决于你当时提问的方式，Agent 也只能凭经验猜你想要的风格。

第二层：调一个 Skill（Skill Call）

当你想要的是某种特定处理结果，而不是一个「差不多就行」的答案，就该想到 Skill。

例子 1：你看到一篇行业报告，术语满天飞，想发给完全外行的朋友看。

随口问可能得到一份「精简版」，但还是硬邦邦的。如果你调 ljg-plain，它的工作方式本来就是「把复杂内容讲成普通人能懂的话」——你会得到一份更像「朋友口述」的版本。

例子 2：你想研究「为什么现在的年轻人不爱换手机了」，刷到一堆零散分析，脑子很乱。

随口丢一句，Agent 可能给你罗列十条原因，但你还是抓不到重点。如果你调 ljg-rank，它专治的就是「把现象砍到剩几根硬支柱」——帮你找出真正撑着这个问题的 3-4 股劲儿。

例子 3：你读完一本书，想把一个打动你的观点做成可以发朋友圈的图。

随口让它写「帮我写段文案」，它可能写得很长、格式不对、也不像卡片。如果你调 ljg-card，它会直接按做卡片的逻辑来组织文字：句子短、有重点、适合视觉呈现。

第三层：把多个 Skills 串成 Workflow

有时候一件事不是一步能搞定的。

比如你读到一篇很难的论文，想最后做成一张科普卡片发朋友圈。那你的 Workflow 可能是：

ljg-rank
→
ljg-plain
→
ljg-card

这就不是「调一个命令」了，而是你心里有了一条从小路到大道的高速公路。

Skill 的真正价值：减少来回试错的次数

很多人觉得 Skill 「很酷」或者「很高级」。其实不是。

它最大的价值是省口水、稳预期。你不用再从零跟 Agent 解释「我要你简短一点、口语一点、分点来说」——Skill 已经把这些偏好内化了。你就像是请了一位已经熟悉你风格的助手，而不是每次都要重新培训。

什么时候该想到「要不要调个 Skill」

这里给你一个轻量判断清单。如果下面任何一条中了，就值得想一想：

  • 这个问题我上周好像也以类似方式问过 Agent。
  • 我想要的不只是「答案」，而是已经收拾好的样子（简化版、骨架版、卡片版）。
  • 我预感要跟 Agent 来回三四轮才能说清楚要求。
  • 我想把这件事变成一个以后能重复用的 habit。

不用全中，中一条就够了。

本课小结

Skill 不是咒语，不是更厉害的 prompt 模板。

它是：让 Agent 跟着你已经试顺、信得过的一套 habit 来干活。

随口问是散步，调 Skill 是开车走高速，串 Workflow 是一次完整的旅行规划。后面我们要学的，就是怎么看清路况、选对高速、偶尔自己拼一条路线。
