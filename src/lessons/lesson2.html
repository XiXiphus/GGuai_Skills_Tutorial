Lesson 02

Prompt 五维 · 开口成章

不是 prompt 越长越好，是这五维越清楚越好。

一个常见的失败

想象你对伴侣说：「帮我做顿好吃的。」

对方可能会回：「你想吃什么菜系？给谁吃？几点要？有没有忌口？」

——不是他不愿意做，是你给的方向太宽，他反而要猜。

跟 Agent 说话也是一样。一句「帮我分析一下」听起来简单，但 Agent 不知道你手里有什么材料、想要什么样的分析、说给谁听、多长、什么风格。结果就是：它猜一个方向，你觉得「不是这个感觉」，来来回回改好几轮。

这课要学的是：怎么在第一次开口时，就把该交代的交代清楚。

五维：材料、目标、对象、限制、成稿长啥样

不用背，理解就行。

1. 材料（Material）

问的是：你手里有什么？

  • 坏例子：「分析一下这个现象」
  • 好例子：「我有一段访谈录音的文字稿，想请你帮我梳理」

材料说清楚，Agent 才知道从哪开始。是一段话、一个链接、一个文件、还是什么都没有只靠你描述？这决定了它怎么帮你。

2. 目标（Goal）

问的是：你想变成什么？

  • 坏例子：「帮我整理一下」
  • 好例子：「帮我把这段话变成一个我能发给朋友的解释」

「整理」太抽象。「变成什么」具体——是一篇总结、一段对话、一张卡片、一个判断？

3. 对象（Audience）

问的是：这最终说给谁听？

  • 坏例子：（没说）
  • 好例子：「给完全不懂这个话题的人看」或「给同行看，不用解释基础概念」

对象决定语气、深度、用不用术语。Agent 需要知道这个，才能选对说话方式。

4. 限制（Constraints）

问的是：有什么不能碰的边界？

  • 坏例子：（没说，结果 Agent 写了一千字，你只需要两百）
  • 好例子：「控制在 300 字以内」「不要用专业术语」「不要说得太绝对」

限制不是为难 Agent，是帮它收窄范围，省得你后面再删。

5. 成稿长啥样（Format）

问的是：你最后要拿到什么 format 的东西？

  • 坏例子：「分析一下」
  • 好例子：「给我一个可以复制粘贴的文案」或「生成一张卡片用的文字」

这一点说清楚，一次到位，不用再来回改 format。

一个能直接用的开口模板

下次想说「帮我弄一下」的时候，换成这个结构：

我有 [材料]，想请你帮我 [目标]，说给 [对象] 听，最后要 [成稿长啥样]，注意 [限制]。

填空示例：

我有一段关于睡眠研究的摘要，想请你帮我提取核心结论，说给刚入门的人听，要一张能分享的卡片文案，注意不要用太多医学名词。

这就是一句靠谱的 request。

这一课的核心

你不是在学「怎么写 prompt」，你是在学：怎么把自己真正想要的东西，拆成别人能听懂的几句话。

这个能力，对 Agent 有用，对和人沟通也一样有用。
